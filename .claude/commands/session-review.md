Проанализируй работу с AI за неделю на основе журнала сессий.

По умолчанию — текущая неделя. Если передан аргумент (например `W11` или `2026-W11`) — указанная неделя.

## Инструкция

1. Определи номер недели: `date '+%Y-W%V'`. Прочитай `Resources/Session Log/YYYY-WNN.md`.
2. Если файл не найден или пуст — сообщи и остановись.
3. Разбери все записи (`### YYYY-MM-DD HH:MM`). Для каждой извлеки:
   - Дату и время
   - Строку **Сессия:** (метрики)
   - Строку **Контекст:** (домен)
   - Пункты работы
   - Wikilinks на проекты
4. Посчитай статистику и выведи отчёт.

## Формат отчёта

```
## Обзор недели YYYY-WNN (ДД.ММ — ДД.ММ)

### Сводка
- Сессий: N (обновлённых: M)
- Промптов: ~X (сумма из строк Сессия:)
- Активное время с AI: ~Xч Yмин
- Дней с сессиями: N из 7
- Локации: [список]

### Распределение по доменам

| Домен | Сессий | Ключевые результаты |
|-------|--------|---------------------|
| Компания/продукт | 5 | ... |
| LifeOS/инфраструктура | 3 | ... |

### Распределение по дням

| День | Сессий | Активное время | Домены |
|------|--------|---------------|--------|
| Пн 17.03 | 3 | ~1ч 20мин | Компания, LifeOS |
| Вт 18.03 | 0 | — | — |

### Паттерны

- **Пиковые часы:** [утро/день/вечер]
- **Средняя сессия:** ~X промптов, ~Y мин активного времени
- **Параллельная работа:** N сессий с параллельными (из M всего)
- **Самый нагруженный день:** [день, N сессий]
- **Фокус или разброс:** [один доминирующий домен / равномерно / хаотично]

### Топ-3 результата недели

1. [Самый значимый результат — артефакт, решение, навык, процесс]
2. ...
3. ...

### Доля AI в работе

Активное время с AI за неделю: ~Xч Yмин.
```

## Правила

- Отчёт — зеркало, не коуч. Факты без советов.
- Метрики из строк **Сессия:**. Записи без метрик — считать только количественно.
- Домены из строк **Контекст:**. Без контекста — классифицировать по wikilinks и содержанию.
- Если записи в старом формате (без Сессия:/Контекст:) — работать с тем что есть.
- Топ-3 — суть: что появилось, а не пересказ.
