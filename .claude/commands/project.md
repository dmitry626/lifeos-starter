Управление проектами. $ARGUMENTS

## Подкоманды

Разбери `$ARGUMENTS`:
- `new` — создание проекта. Остаток — название/тема.
- `plan` — доработка проекта. Остаток — project_id или название.
- `work` — загрузка контекста для работы. Остаток — project_id или название.
- `archive` — архивация проекта. Остаток — project_id или название.
- Пусто — покажи справку.

---

## Общие правила

- Русский язык, без эмодзи.
- Ссылки: `[[wikilink]]`, без `.md`.
- Реестр проектов: `01-Projects/README.md`.
- Шаблон: `Templates/Project Template.md`.
- Архив: `99-Archive/`.

## /project new

### Что делает
Создаёт новый проект: папку, README, предстартовый анализ.

### Алгоритм

1. **ID.** Прочитай `01-Projects/README.md`, возьми `last_project_id`. Если 0 — первый проект будет `001`. Формат: `NNN` (трёхзначный).

2. **Название.** Из аргументов или спроси.

3. **Создать структуру:**
   - `01-Projects/NNN Название/README.md` — по шаблону
   - `01-Projects/NNN Название/Artifacts/` — пустая папка

4. **Предстартовый анализ:**
   - Поиск по vault: связанные заметки, документы (Grep/Glob)
   - Оценка: зачем проект, какие риски, что уже есть
   - Записать в `Предстартовый анализ (Kickoff).md`

5. **Обновить реестр:** добавить строку, обновить `last_project_id`.

### Шаблон Kickoff

```markdown
---
Дата: YYYY-MM-DD
tags:
  - artifact
---

## Что найдено в базе знаний
- Заметки, документы по теме (со ссылками)

## Предложения по проекту
- Рекомендации

## Открытые вопросы
- Что стоит прояснить перед стартом

## Риски
- Что может помешать
```

## /project plan

### Что делает
Итеративная доработка проекта: ревью README, вопросы, обновление.

### Алгоритм
1. Найти проект по ID или названию.
2. Прочитать README.
3. Проверить полноту: цель, DoD, план, связи.
4. Задать уточняющие вопросы.
5. Обновить README по ответам.

---

## /project work

### Что делает
Загружает контекст проекта для работы в сессии.

### Алгоритм
1. Найти проект, прочитать README.
2. Прочитать CLAUDE.md проекта (если есть).
3. Показать: статус, открытые задачи, блокеры.
4. Список артефактов и встреч (последние по дате).

Вывод — минимальный (3-5 строк): статус, что в работе, блокеры.

---

## /project archive

### Что делает
Закрывает проект: извлечение знаний, перенос в архив.

### Алгоритм
1. Найти проект, прочитать README.
2. Проверить DoD — все ли выполнено.
3. **Спросить подтверждение** — не архивировать без двойного подтверждения.
4. Извлечь переносимые знания (что узнали, что можно переиспользовать).
5. Обновить статус: `5-done`, заполнить `done:`.
6. Переместить папку в `99-Archive/`.
7. Обновить реестр.
