Обработай встречу. $ARGUMENTS

## Назначение

Декомпозиция встречи на строительные блоки: решения, задачи, знания, идеи, риски. Связывание с vault: люди, проекты. Предложение следующих действий.

## Вход

$ARGUMENTS содержит путь к файлу встречи (`.md` или `.txt`) или описание/заметки.

Если $ARGUMENTS пуст — спроси пользователя: "Какую встречу обработать? Можешь скинуть файл, вставить текст заметок или просто рассказать, что обсуждали."

Стенограмма (`.txt`) — если рядом с `.md` файлом лежит файл с тем же именем и суффиксом `(стенограмма).txt`, прочитай его тоже.

## Модель документов

| Файл | Статус |
|------|--------|
| Исходный файл (summary/стенограмма) | Read-only. Не редактировать. |
| Файл анализа `(анализ).md` | Создать или обновить |

Файл анализа создается рядом с исходным файлом, с тем же именем + суффикс `(анализ)`.

## Алгоритм

### Фаза 1 — Сбор контекста

1. Прочитай файл встречи (summary и/или стенограмму).
2. Извлеки список участников. Поищи файлы по vault (glob `**/<Имя>.md`).
3. Прочитай `01-Projects/README.md` — список активных проектов.
4. Определи бизнес-контекст по содержимому.

### Фаза 2 — Декомпозиция

Из текста извлеки:

- **Решения** — что конкретно решили, кто решил, почему.
- **Задачи** — действия с ответственным и сроком. Формат: `* [ ] Задача (Ответственный: Имя, Срок: дата или "не указан")`
- **Знания** — переносимые принципы, паттерны (ценные вне этой встречи).
- **Идеи** — гипотезы, предложения для проработки.
- **Открытые вопросы** — неразрешённые темы.
- **Риски** — угрозы, зависимости, ограничения.
- **Факты и метрики** — числа, даты, KPI.

### Фаза 3 — Связывание

Для каждого блока:
- **Люди**: `[[Имя Фамилия]]` (если файл найден в vault)
- **Проекты**: `[[01-Projects/<проект>/README|<имя>]]`
- **Документы vault**: ссылки на связанные файлы

### Фаза 4 — Маршрутизация

Где встреча должна жить:
1. `01-Projects/<проект>/Meetings/` — если привязана к проекту
2. `Resources/Встречи/` — по умолчанию (разовая, внешняя)

Предложи маршрут, не перемещай без подтверждения.

## Формат файла анализа

```
---
tags: meeting-analysis
Дата анализа: YYYY-MM-DD
---

## Контекст

* Участники: [[Имя1]], Имя2
* Связанные проекты: [[проект|имя]] или "не выявлены"

## Решения

* Решение 1 — контекст

## Задачи

* [ ] Задача 1 (Ответственный: Имя, Срок: дата)

## Знания

* Принцип/паттерн 1

## Идеи

* Идея 1

## Открытые вопросы

* Вопрос 1

## Риски

* Риск 1

## Факты и метрики

* Факт 1

___
**Источники:**
- [[<имя файла встречи>]]
```

## Вывод в чат

После создания файла — краткая сводка + предложения:

1. **Маршрутизация** — переместить файл?
2. **Создать задачи** — если подключен задачник, предложить создать задачи
3. **Обновить проект** — если решения привязаны к проекту
4. **Создать карточку персоны** — если упомянуты новые люди

Каждое предложение — отдельно. Выполнять только после подтверждения.

## Правила

- Русский язык.
- Без эмодзи.
- Краткость: блоки — 1-3 предложения.
- Не додумывать. Если не ясно — "требует уточнения".
- Короткая встреча = короткий анализ. Не раздувать.
