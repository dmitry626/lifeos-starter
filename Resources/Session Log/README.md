# Журнал сессий

Операционный лог работы с Claude Code. Понедельные файлы (ISO 8601, Пн–Вс), обратная хронология.

## Как это работает

```
session-tracker (хук)      автоматический, фоновый
       │                   пишет в /tmp/claude-session-{id}.log
       ▼
session-analyze (скрипт)   вызывается по запросу
       │                   читает /tmp, считает метрики
       ▼
/session-log (навык)       ручной вызов
       │                   пишет в vault (эта папка)
       ▼
/session-review (навык)    ручной вызов
                           читает понедельные файлы, показывает паттерны
```

## Структура файлов

Один файл на неделю: `YYYY-WNN.md` (например `2026-W12.md`).

Неделя начинается в понедельник, заканчивается в воскресенье (ISO 8601).

## Трекинг сессий

Хук `UserPromptSubmit` автоматически логирует timestamp каждого промпта в `/tmp/claude-session-{session_id}.log`. Формат: ISO 8601 с часовым поясом (`2026-03-21T08:14:59+0800`).

Скрипты:
- `scripts/session-tracker.sh` — хук, пишет timestamp при каждом промпте
- `scripts/session-analyze.sh` — анализатор: метрики сессии

Конфигурация хука: `.claude/settings.json`

## Формат записи

```markdown
### YYYY-MM-DD HH:MM

**Сессия:** начало HH:MM, промптов N, активное время ~Xч Yмин, пауз: K | параллельных сессий: M | локация
**Контекст:** [домен/область]
<!-- session: SESSION_ID -->

- Что сделано и зачем. Wikilinks на проекты.

Изменено: `файл1`, `файл2`
```

## Что означают метрики

- **Активное время** — сумма интервалов между промптами короче 15 минут. Реальное время вовлечённой работы.
- **Паузы** — интервалы длиннее 15 минут (обед, встреча, перерыв). Не считаются активным временем.
- **Параллельные сессии** — другие сессии Claude Code, у которых были промпты в период текущей сессии. Факт, не оценка.
- **Локация** — определяется по часовому поясу системы в момент промпта. Смена в ходе сессии = перемещение.

## Навыки

- `/session-log` — вызывать перед паузой или завершением блока работы. Повторные вызовы обновляют запись.
- `/session-review` — сводка недели: сессии, домены, паттерны, топ результатов.

## Настройка

В `scripts/session-analyze.sh` можно изменить:
- `PAUSE_THRESHOLD` (по умолчанию 900 секунд = 15 минут)
- Функцию `tz_to_location()` — маппинг UTC offset на ваши локации
