# Docs -- области фокуса и ответственности

Здесь живёт всё, что требует постоянного внимания и не имеет конечной даты: бизнес, финансы, здоровье, семья, развитие. Отличие от проектов: проект имеет цель и конец ("запустить продукт к марту"). Область ответственности -- бессрочна ("управлять компанией", "следить за здоровьем").

## Как организовать

Каждая область -- отдельная папка с `README.md` внутри. Количество областей не фиксировано: у кого-то 5, у кого-то 15. Создавай папку, когда появляется второй файл по теме.

**Типичные области для предпринимателя/руководителя:**

| Область | Что внутри |
|---------|-----------|
| **Компания** (по одной на каждый бизнес) | Стратегия, продукты, команда, координации, журнал решений |
| **Финансы** | Бюджет, счета, налоги, инвестиции, стратегия |
| **Здоровье** | Чек-апы, анализы, врачи, прививки, питание |
| **Семья** | Дети, имущество, документы, персонал |
| **Личное развитие** | Цели, привычки, коучинг, книги, рефлексия |
| **Хобби** | Спорт, музыка, путешествия, увлечения |

Это примеры, не обязательный список. У тебя могут быть другие области: Социум, Благотворительность, Образование -- что угодно, что требует постоянного внимания.

## Зачем это нужно AI-ассистенту

Claude Code видит файлы в vault и использует их как контекст. Чем точнее описаны области ответственности, тем релевантнее ответы. Без контекста компании любой совет будет generic. С контекстом -- конкретный.

## Принципы

- **Факты, не мнения.** Docs/ -- это "что есть": описания, процедуры, контакты, метрики.
- **Один файл -- одно место.** Если тема пересекается (здоровье детей -- Семья или Здоровье?), файл живёт в одном месте, из другого -- `[[ссылка]]`.
- **Не создавай пустые папки.** Папка появляется, когда есть что в неё положить.
